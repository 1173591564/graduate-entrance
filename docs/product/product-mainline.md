# 使用逻辑主线设计（AI 赋能考研 · 自用版）

> 本文回答一个问题：**当前功能已经很全，但缺一条清晰的"使用逻辑主线"，它应该是什么样？**
>
> 定位：单人自用工具，目标是**用人机分工协作达成初始 360 分**。它不是"待办 App"，而是**你和一个军师共同打一场仗**。
>
> 与 `11408备考系统设计.md`（讲架构/技术栈/数据模型）互补：本文只讲**产品的因果闭环与人机分工**，不重复架构细节。

---

## 0. 一句话产品承诺（主线的锚）

> **"告诉我考试、可用时间和目标分数；每天我告诉你今天到底学什么；每周根据实际发生帮你调整。用户只负责学，计划 / 整理 / 分析全交给 AI。"**

整个系统只服务这一句话。任何功能，如果不能被这句话串起来，就是游离的孤岛。

---

## 1. 问题诊断：功能齐全，但各自为政

现状是"feature silos"——大纲树、任务池、计划、Today、打卡、顺延/请假、复盘、SM-2、拍题/识别、掌握度、边际注记 都在，但缺一条把它们串成因果的**闭环叙事**。用户打开 App 时答不出"我现在该干嘛、为什么是这件事"。

主线断掉的根因：
- **Today 还不是唯一重心**：入口分散（聊天式首页），注意力被稀释。
- **万物没有挂 KP、也不回写掌握度**：任务完成 / 错题 / 复习结果没有真正驱动计划，掌握度更像展示指标。
- **复盘像报表，不回写计划**：统计沦为虚荣指标，没有变成"下周怎么改"。
- **复习 / 拍题不回流到每日**：各自独立成页，没进 Today。
- **系统被动**：只处理用户喂进来的数据，不主动采集、不主动预警。

---

## 2. 主线 = 一个反馈闭环 × 三个时间尺度

```
三个数字（考什么+日期 / 每天可用时间 / 目标分数）
        │
        ▼
  AI 反推：目标分 → 各科分配 → KP 掌握目标 → 按性价比排路径
        │            （数据在系统侧：真题分值分布 / 大纲权重 / 教材章节）
        ▼
  校准期（前 3–5 天）：用真实表现反推"现状"，修正计划
        │
        ▼
┌──────────────► Today 今日一屏（唯一入口）◄──────────────┐
│               顺境上强度 / 逆境给最小一步                 │
│                        │                                 │
│                        ▼                                 │
│        执行：番茄钟(自动记时) · 打卡 · 拍题问AI · 到期复习 │
│                        │                                 │
│                        ▼                                 │
│         三信号：打卡(投入) · 对话(困惑) · 复习小测(掌握)  │
│                        │                                 │
│                        ▼                                 │
│              更新掌握度 / 覆盖度 / 薄弱 KP               │
│                        │                                 │
│         每日无感微调 ◄──┤                                 │
│                        ▼                                 │
│      每周大复盘：diff（前→后+理由+影响）→ 用户一键拍板    │
└────────────────────────┴─────────────────────────────────┘
        │
        ▼
  写回下一轮 · 倒计时+覆盖/掌握进度条 见证推进
```

三个时间尺度：

- **战略层（一次性，偶尔改）**：目标 + 科目 + 目标分数 + 可用时间 → 宏观蓝图（阶段 / 科目配比 / KP 覆盖与掌握目标）。对应现有：syllabus import、phase、availability、task-pool。缺口：对用户"隐形"，应有一个"这是我的备考蓝图"视图。
- **每日循环（天天，真正的主线）**：打开即落地 Today → 番茄钟做 → 打卡实际耗时 → 拍错题 → 到期复习卡也在这里。一屏回答"现在干嘛、为什么"。对应：Today、Pomodoro、打卡、capture、reviews。
- **每周循环（每周）**：Retro：实际 vs 计划、掌握/覆盖变化、错题健康度 → 产出可确认的"计划改动 diff" → 写回下周。对应：weekly-stats、retro、reschedule、leave。

## 3. 输入：三个数字 + 一次选择，其余全靠系统

用户**只填三样**（尽量从选项里选，不手打）：

1. **考什么 + 考试日期**（倒计时）
2. **每天可用时间**
3. **目标分数**（初始 360）

外加一次选择：**用什么教材/课程**（张宇 / 王道 …），从选项里选。

**"数据驱动"的数据绝大部分在系统侧**，用户一个都不用填：
- 历年真题的题型分值分布（反推 KP 掌握目标）
- 大纲 KP 权重
- 教材章节结构（张宇 30 讲、王道各章 → KP 映射）
- 时间预算数学

### 目标分数是最好的锚（为分数做反向工程）

- 目标总分 → 各科分目标（按薄弱/性价比分配，**不是平均**）
- 各科分目标 →（真题题型分值分布）→ 反推"哪些 KP 必须掌握到什么程度"
- KP 目标 + 每天时间 → 排出到考试的路径

360 的含义：**不需要门门顶尖，而是合理分配 + 性价比最大化**——这恰是 AI 最擅长、人最容易算错的（人凭感觉容易平均用力或死磕难点）。

### 现状怎么来：不问，靠推

用户不填"当前水平"。前 3–5 天当**校准期**，用真实完成速度 + 首轮复习表现反推起点，之后每天的信号持续修正。**现状是被"学"出来的，不是被"填"出来的。**

---

## 4. 人机分工（本产品的灵魂）

按"能不能外包"三分：

### ① 只有你能做（绝不外包）
- 真正的学习——把知识装进脑子
- 诚实反馈——打卡耗时、复习评级、不懂就问
- 大方向拍板——接受/拒绝复盘 diff

### ② 必须全甩给 AI（你一秒都别花）
- 排计划、算性价比、反推 KP 目标、关联知识点
- 算掌握度、定复习时间、组织错题、每日/每周调整

### ③ 协作区（AI 提，你拍）
- 现状判断：AI 推 + 你纠
- 目标可行性：AI 算 + 你拍板

### 冲 360，AI 的四个杀手锏
1. **反推分配**：合理分配 + 性价比最大化。
2. **性价比守门**：持续把你从低性价比里拽出来（"这道极难题对 360 没用，跳过"）。
3. **记忆管理**：SRS + 错题调度全外包，你只管"到点就复习"。
4. **诚实的进度镜子**：三信号 + 可行性预警，让你早调整，别到最后才发现来不及。

### 自用场景的特有风险：自欺
没有外部监督，最大的敌人是给自己打卡注水、评级虚高、逃避薄弱科。**信号质量决定 AI 质量，垃圾进垃圾出。** 系统要刻意对抗自欺：复习尽量用客观小测代替自评，可行性预警敢说难听真话。**AI 一半是军师，一半是不留情面的镜子。**

---

## 5. 三个信号：现状的"三角测量"

| 信号 | 反映 | 特点 | 采集方式 |
|---|---|---|---|
| **打卡** | 投入 / 完成度（学了没、花多久） | 硬、便宜、天天有；只有量没有质 | 番茄钟自动记时，不手填 |
| **提问对话** | 理解 / 困惑（暴露薄弱 KP） | 自然产生，比自评真实；但测不到盲区 | 用户本来就要问，顺便采集 |
| **复习 / 小测** | 客观掌握 + 补盲区 | 能测出用户没察觉的洞 | SM-2 复习 + 主动插入小测 |

三者互补，缺一角就测不准（缺对话只有量没有质，缺主动检测永远测不到盲区）。

### 关键原则：信号必须是"学习的副产品"
绝不出现"请给这个知识点打分""请给这题打标签"这类**为系统而存在的操作**。一旦出现，"用户只负责学"就破功。

### 对话必须"结构化回流"（否则聊完就蒸发）
- 给 AI **助教/军师**角色约束，不做开放陪聊
- 每轮对话：AI 自动标注涉及 KP → 判断理解程度 → 一键"记为错题 / 加入复习 / 生成即时检测题"
- **拍题就是"对话"的特化**（拿题来问），收成同一个"问 AI"入口，走同一条回流链路

## 6. 主动性：系统自己去挖，而不是等你喂

"用户啥都不填"能成立的前提，是系统**主动补上**你没填的部分。主动性不是加表单，而是让系统主动采集/推断。

### 四级主动性（从无感到略打扰，越往下越克制）
- **L1 主动观察（零打扰，默认全开）**：从行为里挖——耗时、评级、提问、番茄钟中断、打卡时间点、拖延时段……**能观察到的绝不开口问。**
- **L2 主动分析 + 预警（克制推送）**：主动发现"某 KP 掉了 / 某科拖后腿 / 360 亮红灯"并提醒。这是军师的主动性——替你操心你没意识到的风险。
- **L3 主动提问校准（稀少，高价值时一句）**：关键时刻问一句而非填表（"这章吃力吗？""是概念没懂还是计算错？"），一次一句、可跳过。
- **L4 主动检测盲区（伪装成学习）**：合适时机插入小测/回顾题，探测你不会主动来问的盲区（unknown unknowns），小测本身就是有效学习。

### 红线：主动 ≠ 打扰
能观察就别问；要问就稀少+高价值+可跳过；检测伪装成学习；预警只推真正需要行动的。

### 长期记忆红利（自用场景）
主动性质量取决于对用户画像的深度（哪科快/慢、易错模式、拖延时段、注水倾向）。长期单人使用，画像可积累得很深，主动得越来越准——这是自用相比通用产品的最大优势。

## 7. 教材映射层：精准落地（KP 保通用，教材保落地）

用户用张宇（数学）、王道（408）。把计划落到具体章节/讲次，精准度大跳跃：
- **可执行性**：任务从"学极限"变成"看张宇基础班第 X 讲 + 做对应 1000 题"
- **时间预算更准**：AI 知道还剩几讲、每讲多久
- **性价比先验更细**：章节对应真题分值/难度，是现成数据
- **错题/复习定位精确**：错题挂到"张宇第 X 讲例 3"

### 三层架构（换教材不崩）
1. **底层：KP + 掌握度**（教材无关，稳定）
2. **中层：教材/课程 → KP 映射**（可插拔）
3. **上层：Today 任务 = 具体讲次/章节**（用户看到的）

> KP 决定"学什么到什么程度"，教材映射决定"具体翻哪本的哪一讲"——前者保通用，后者保落地。
>
> 注：自用场景，教材版本/年份维护成本可以忽略；但仍保持"教材是可插拔皮肤、KP 才是底层真相"的分层，避免被某套教辅锁死。

## 8. 逆境机制：系统默默吸收生活的混乱

决定产品生死的不是顺境，而是**计划被现实打碎时怎么办**。既然"用户只负责学"，连"我落后了"的焦虑都该由系统吸收。

### 核心重构：计划是"活的建议"，不是"契约"
弱化"落后/欠债"这个概念本身。计划每天基于三信号**重新优化**，永远只回答一句："**按你现在的真实进度和剩余时间，今天最划算的事是什么。**" 昨天没做完的不是债，是进入下一轮性价比重排的候选。时间变少了，AI **该砍低性价比、保高分项**，而不是无脑顺延（无脑顺延会雪崩）。

### 三种"不顺利"各有优雅应对
1. **落后**：AI 重排 + 保大分，必要时降覆盖广度换重点深度，并**说明取舍**（"为保数学大题，英语作文顺延"）。
2. **状态差（低能量日）**：见下。
3. **中断（消失几天回来）**：不甩逾期，而是"欢迎回来，已按剩余时间重排，从这里开始"。

### 低能量日 / 最小可执行一步（核心洞察：换类型，不只是减量）
- **本质是切到低认知负荷任务**：状态差时给"啃新难点"必败；应切到复习旧的、巩固已会、整理错题。
- **AI 挑"那一件"的低能量日专用排序**：① 今天到期的复习（有硬窗口、低负荷、高性价比）② 高性价比 + 低难度任务 ③ 明确排除高难度新知识。即在（到期复习 ∪ 高性价比任务）里挑**认知负荷最低 + 时间最短**那个。
- **触发**：以用户主动声明"今天状态不好"为主；AI 被动推断（完成率下滑/打卡变晚/耗时超标/番茄钟频繁中断）只做**温和提示**，不强制。
- **行为设计红线**：显示一句"就做这一件：X，约 15 分钟"+ 开始按钮，降低启动阻力；**完成标准下调**——哪怕只做 10 分钟也算"今天保住了"，**绝不计入落后、绝不产生欠债**，做完给正反馈保住连续性。
- **信号 + 反面陷阱**：偶尔低能量→吸收掉不反应；持续低能量→留给周复盘下调目标/时间。别让轻量模式变逃避后门——天天点，AI 要在复盘温和指出"这周 5 天轻量，是目标太高还是时间不够？"

## 9. 复盘：闭环真正合龙的地方

分析必须变成计划改动写回去，否则复盘只是报表。**大复盘要能在 1–2 分钟看完并拍板。**

### 每日 vs 每周分层
- **每日 = 无感微调**（战术层，系统自己扛，不打扰）
- **每周 = 需确认的方向调整**（战略层：动配比/目标/砍补，才找用户）
- **小事系统扛，大事（动承诺）才找用户** —— "用户只负责学"和"用户掌控方向"同时成立。

### 每周 diff 三段式（不堆报表）
1. **本周实况**（读秒级）：只推"最值得注意的 3 件事"（最大偏差），其余折叠。
2. **归因**（说人话）：不是"完成率 62%"，而是"数学落后主因是极限那章耗时翻倍；英语反而超前"。
3. **建议改动 diff**（核心）：顺延 / 砍项 / 补救 / 配比调整 / 目标可行性预警。

### 每张卡片"三件套"
> **改什么**（前→后）· **为什么**（挂到具体信号/数据）· **影响**（接受后下周会怎样）

例：
> 📌 数学配比 30% → 38%
> 因为：极限、中值定理两个 KP 掌握度掉到目标线下，错题密度本周最高
> 影响：下周数学 +3 个补救任务，英语新课顺延 2 天

### 拍板粒度
- 默认**一键全接受**（别逼用户逐条审）
- 可展开逐条微调/拒绝（掌控感）
- **拒绝也是信号**：AI 记住偏好，下次少提这类
- **动"承诺"的决策（降目标）必须用户显式选**，系统绝不默默降
- diff **可回滚**，降低拍板心理成本

### 健康度红线
顺利的周该是"这周很好，微调两处，一键确认"，不该每周甩一堆改动制造焦虑。**改动量是系统健康的体温计。**

---

## 10. 北极星与进度锚

持久可见的"**距考试 X 天 + 覆盖度/掌握度进度条 + 预测分数 vs 目标 360**"。边际注记已有倒计时，把它升级成主线的进度锚——用户每天的努力都能看到它在动，这就是坚持的动机闭环。

## 11. 现有功能映射与缺口

| 主线环节 | 现有支撑 | 缺口 |
|---|---|---|
| 三数字输入 | availability、phase、目标 | 缺"目标分数→各科→KP 目标"反推；输入未收敛成 3 项 |
| 反推 KP 目标 | syllabus/KP 树、task-pool | 缺真题分值分布数据 + 性价比排序引擎 |
| 教材映射层 | KP 树 | 缺 教材章节→KP 映射、章节级任务 |
| 校准期 | — | 缺冷启动校准逻辑 |
| Today 唯一入口 | Today、Pomodoro、打卡 | 到期复习/待识别错题未并入 Today；首页仍分散 |
| 三信号 | 打卡、SM-2、拍题 | 缺"对话"信号 + 对话结构化回流；掌握度未被三信号真正驱动 |
| 主动性 | APScheduler（生成每日任务） | 缺 L2 预警 / L3 主动提问 / L4 主动检测；缺用户画像/长期记忆 |
| 逆境机制 | reschedule、leave | 缺"活计划重优化（砍项/保大分）"、低能量日/最小一步 |
| 复盘 diff | retro、weekly-stats、reschedule | 复盘未产出可确认 diff、未一键写回、无可回滚 |
| 进度锚 | 边际注记倒计时 | 缺覆盖/掌握进度 + 预测分 vs 360 |

## 12. 分阶段接线 Roadmap（不改大架构，先接线）

**Phase 1 — 把 Today 变成唯一重心**
1. Today 设为默认入口；到期复习数 + 待识别错题并入 Today。
2. 首页顶部常驻"倒计时 + 覆盖/掌握进度 + 预测分 vs 360"。

**Phase 2 — 让掌握度真正被驱动（系统"活"起来）**
3. 任务完成 → 回写对应 KP 掌握度；拍题 → 关联 KP。
4. 三信号加权算掌握度；掉下去的 KP 自动加补救任务/提前复习。

**Phase 3 — 目标分数反向工程 + 教材落地**
5. 接入真题分值分布；"目标分→各科→KP 目标"反推 + 性价比排序。
6. 教材章节→KP 映射，Today 任务落到具体讲次/章节。
7. 冷启动校准期逻辑。

**Phase 4 — 主动性**
8. L2 预警（掌握度掉/进度偏/可行性变）。
9. "问 AI"统一入口 + 对话结构化回流（L3/L4 主动提问与检测）。
10. 用户画像/长期记忆。

**Phase 5 — 闭环合龙**
11. 逆境机制：活计划重优化 + 低能量日/最小一步。
12. 每周复盘产出可确认、可回滚的 diff，一键写回（复用 reschedule/leave）。

---

## 13. 一条贯穿始终的设计原则

**每个核心对象都要能回答四个问题：**
- 它代表什么？
- 它从哪来？
- 完成/改变它影响什么？
- 结果流向哪里？

例：
- 任务 → 关联 KP + 阶段 + 教材讲次
- 错题 → 关联 KP + 复习计划
- 复习评级 → 影响掌握度 + 下次到期
- 完成的任务 → 影响真实节奏 + 未来容量
- 复盘建议 → 变成可确认的计划改动

**能回答这四问，孤岛就连成了闭环。**
